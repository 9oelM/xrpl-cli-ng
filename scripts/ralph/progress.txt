## Codebase Patterns
- Test framework: vitest with ESM imports (use `import ... from './module.js'` with .js extension)
- TypeScript strict mode with NodeNext moduleResolution — always use .js extensions in imports
- Existing commands use `createInterface` from readline for password prompts (no masking)
- To mock process.stdin in tests: use Object.defineProperty with a custom EventEmitter having isTTY/setRawMode/resume/pause/setEncoding mock fns
- process.exit tests: mock with a fn that throws a sentinel Error, await rejects.toThrow()
- For sequential async prompts in tests, use nested setTimeout(0) to allow microtasks between data emissions

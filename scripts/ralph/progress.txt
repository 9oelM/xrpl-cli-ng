## Codebase Patterns
- `resolveAccount(input, keystoreDir)` returns the input unchanged if it looks like an XRPL address (`/^r[a-zA-Z0-9]{24,33}$/`); otherwise scans keystore for an alias match and throws if not found. Always wrap in try/catch when you need a clean "invalid address" error.
- Use `--confirm` (boolean flag, default false) for irreversible destructive commands; check `!options.dryRun && !options.confirm` to allow dry-run without confirmation.
- `--no-wait` correct pattern per MEMORY.md: `.option("--no-wait", "...")` with NO default arg; interface property `wait: boolean`; check `!options.wait` for the no-wait path.

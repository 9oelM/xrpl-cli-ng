## Codebase Patterns
- New commands go in `src/commands/<name>.ts`, exported from `src/commands/index.ts`, and registered in `src/index.ts`
- Use `resolveWallet()` helper pattern (see credential.ts) for key material resolution
- `--no-wait` option: use `.option("--no-wait", "...")` (NO third arg), property is `wait: boolean`, check `if (!options.wait)`
- Repeatable CLI options in Commander.js: `.option("--flag <val>", "desc", (val: string, prev: string[]) => [...prev, val], [] as string[])`
- Validation test files should be pure no-network; avoid dummy seeds whose address (e.g. rHb9CJAWyB4rj91VRWn96DkukG4bwdtyTh) conflicts with test signer addresses
- Ordering of CLI validations: check count/structural constraints BEFORE per-item validation (e.g., check >32 signers before validating each address)

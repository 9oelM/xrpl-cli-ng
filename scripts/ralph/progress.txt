## Codebase Patterns
- Commands follow payment.ts pattern: walletFromSeed helper, key material validation (exactly one of --seed/--mnemonic/--account), withClient wrapper, autofill+sign+submitAndWait flow
- TrustSet flags are in `TrustSetFlags` from xrpl package (e.g., TrustSetFlags.tfSetNoRipple)
- E2E tests use spawnSync with tsx runner, fund accounts via fundFromFaucet helper
- Currency validation: 3-char ASCII uppercased OR 40-char hex; throw Error with descriptive message
- Subcommands use `new Command('parent').addCommand(new Command('sub').alias('s'))` pattern
- getKeystoreDir(options) + resolveAccount() are used to resolve --issuer and --account
- Commander.js `--no-X` options set `options.X = false` (not `options.noX`); check `options.X === false` to detect the flag; omit the explicit default to allow distinction (default is `true` when not passed)

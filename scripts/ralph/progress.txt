## Codebase Patterns
- Vault E2E tests use devnet (not testnet) — `DEVNET_WS` from `tests/e2e/helpers/devnet.ts`
- Devnet faucet: POST `https://faucet.devnet.rippletest.net/accounts` with `{destination, userAgent}`
- VaultCreate requires elevated fee: `Fee: "200000"` drops
- IOU vault clawback requires: issuer has `asfDefaultRipple` + `asfAllowTrustLineClawback` set BEFORE any trust lines
- Shared issuer + vaultId in beforeAll is an allowed exception for clawback tests (only holders are consumed per test)
- Holders for clawback must be pre-created sequentially in beforeAll (issuer VCB payments can't run concurrently)

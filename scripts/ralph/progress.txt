## Codebase Patterns
- `submitAndReport` helper in check.ts centralizes the submit/wait/json/dry-run flow with an optional extras callback for extra output fields
- Extra output field format: `(label + ":").padEnd(13) + value` (matches escrow.ts fixed-13-char prefix pattern)
- Check ID is extracted from `meta.AffectedNodes[].CreatedNode.LedgerIndex` where `LedgerEntryType === "Check"`
- `convertStringToHex` + `.padEnd(64, "0")` encodes plain string invoice IDs as UInt256
- XRPL epoch conversion: `Math.floor(new Date(iso).getTime()/1000) - 946684800` (create); reverse: `new Date((epoch+946684800)*1000).toISOString()`

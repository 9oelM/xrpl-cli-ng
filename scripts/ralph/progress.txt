## Codebase Patterns
- Oracle PriceDataSeries: each element is `{ PriceData: { BaseAsset, QuoteAsset, AssetPrice?, Scale? } }` — xrpl.js requires this double-wrap
- xrpl.js OracleSet: AssetPrice accepts `number | string`; string must be valid hex. Passing as a JS number is safest for typical oracle values
- xrpl.js validates AssetPrice and Scale must both be present OR both absent in each PriceData object
- `convertStringToHex` from xrpl auto-encodes Provider/AssetClass strings to hex for oracle set transactions
- The `submitTx` helper pattern can be extracted from any existing command file to avoid repetition

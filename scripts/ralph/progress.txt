## Codebase Patterns
- Brokered NFT accept: use fresh/clean accounts for brokered-mode sellers; reusing accounts with prior NFT history can cause `tecOBJECT_NOT_FOUND` due to NFTokenPage state. Use a dedicated wallet (e.g., `broker` who has no prior NFT transactions) as the brokered seller.
- `--offer` repeatable option pattern in Commander.js: `.option("--offer <hex>", "desc", (val, prev: string[]) => prev.concat([val]), [] as string[])` â€” validation must check `options.offer.length === 0` since the default is `[]` not `undefined`.
- `nft_buy_offers` RPC returns `rpcOBJECT_NOT_FOUND` when there are NO buy offers (not just when the NFT doesn't exist); always wrap in try/catch when querying.
- For read-only query commands (`nft offer list`), use `Promise.allSettled` to fetch both sell and buy offers in parallel â€” treat rejected (not found) as empty array.
- `NFTokenCreateOfferFlags.tfSellNFToken = 1` is exported from xrpl.js; use for sell offer flag
- XRPL epoch offset: `XRPL_EPOCH_OFFSET = 946684800` (Unix seconds from Jan 1, 1900 to Jan 1, 2000)
- NFTokenOfferID is in `CreatedNode.LedgerIndex` (not `NewFields`) in AffectedNodes metadata
- For two-wallet E2E tests, call `fundFromFaucet` twice in `Promise.all` â€” `distributeToWallets` has a broken dynamic import (`autofill` is not a function)
- Buy offers: `--owner` required (owner = NFT holder); Sell offers: sender must hold the NFT, no `--owner`
- NFT commands go in `src/commands/nft.ts` with subcommand structure (like offer.ts)
- Use `getNFTokenID(meta as TransactionMetadata)` from xrpl.js to extract NFTokenID; fallback scan AffectedNodes for CreatedNode with LedgerEntryType NFTokenPage
- `convertStringToHex` from xrpl.js converts plain URI strings to hex for NFTokenMint.URI
- `NFTokenMintFlags` enum: tfBurnable=1, tfOnlyXRP=2, tfTransferable=8, tfMutable=16 (never use tfTrustLine=4)
- `resolveWallet` helper pattern is duplicated per command file (no shared utility yet) â€” follow same pattern

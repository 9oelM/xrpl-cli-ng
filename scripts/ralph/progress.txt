## Codebase Patterns
- E2E tests use `spawnSync(TSX, [CLI, ...args])` pattern with 120s timeout per test call
- `wallet import <seed>` takes key material as positional arg, not `--seed`; use `--password`, `--keystore` flags for keystore control
- `account trust-lines --json <address>` returns raw array including `no_ripple`, `quality_in`, `quality_out`, `freeze_peer`, `peer_authorized` fields
- Tests within a describe block run sequentially, so CNY trust line created in test A is visible in test B
- Currency codes available: CNY, JPY, FRZ, AUT (USD, EUR, GBP, CAD, MXN, XYZ used by existing tests)
- For --auth tests: enable RequireAuth on a dedicated authIssuer wallet in beforeAll (not inline in the test) to avoid LastLedgerSequence expiry due to setup time
- Use beforeAll timeout of 240_000ms when funding 3+ wallets + AccountSet setup

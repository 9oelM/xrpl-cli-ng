## Codebase Patterns
- xrpl.js PaymentFlags real values: `tfNoRippleDirect=0x10000`, `tfPartialPayment=0x20000`, `tfLimitQuality=0x40000` (not 0x40000/0x80000 as some docs state)
- Commander.js `--no-X` options: property is camelCase of X (NOT noX), default true, `--no-X` sets it to false; check with `!options.X`; NEVER give explicit `false` default
- Use `@scure/bip39` (`generateMnemonic` + `wordlist` from `@scure/bip39/wordlists/english`) to generate fresh random mnemonics in beforeAll — never use well-known BIP39 test vectors like "abandon...about" on testnet (overused, high sequence number, transaction expiry)
- Fund mnemonic-derived wallets from the faucet-funded sender wallet (via `client.autofill` + `sender.sign` + `client.submitAndWait`) in beforeAll, not directly from the faucet
- For `account info --json` the output is the `account_data` object with `Balance` as a string in drops
- For `account trust-lines --json` the output is an array with `{ account, currency, balance, limit }` — `balance` is positive from the holder's perspective

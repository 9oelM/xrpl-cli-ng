## Codebase Patterns
- EscrowCreate: FinishAfter/CancelAfter must be > current ledger close time (past timestamps fail with tecNO_PERMISSION)
- EscrowCreate: self-escrow (Account == Destination) fails with tecNO_PERMISSION
- EscrowCreate: --condition alone is invalid; xrpl.js requires FinishAfter or CancelAfter even when Condition is set
- isoTimeToRippleTime() from xrpl package handles ISO8601 → XRPL epoch conversion
- Commands with subcommands: create top-level Command("escrow"), add subcommands with .addCommand()
- --no-wait pattern: .option("--no-wait", ...) with NO third arg; interface uses `wait: boolean`; check `if (!options.wait)`
- Offer.ts has reusable resolveWallet() pattern that other commands duplicate

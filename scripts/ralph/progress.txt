## Codebase Patterns
- Tests use `spawnSync(TSX, [CLI, ...args])` with `encoding: "utf-8"` for E2E testing. TSX = `node_modules/.bin/tsx`, CLI = `src/index.ts`
- PATH must be set in env for E2E tests: `/home/vscode/.fnm/node-versions/v22.22.0/installation/bin:${process.env.PATH}`
- Use tmp dirs with `mkdtempSync` + `rmSync(dir, { recursive: true })` in try/finally for test isolation
- getKeystoreDir resolution order: options.keystore → XRPL_KEYSTORE env var → ~/.xrpl/keystore/
- resolveAccount: regex `/^r[a-zA-Z0-9]{24,33}$/` matches XRPL addresses; else scan keystore for label match
- encryptKeystore now takes optional 5th param `label?: string`; included in output only when provided
- All wallet commands import getKeystoreDir from `../../utils/keystore.js` (not local copies)
- Atomic writes use writeFileSync to `<path>.tmp` then renameSync to `<path>` (same pattern for all keystore mutations)
- Alias uniqueness scan: readdirSync + filter `.json` + parse each, skip on error
- Global `--node` option lives on root `program`; subcommand actions get node URL via `getNodeUrl(cmd)` where `cmd` is the last argument passed to the action handler
- `cmd.optsWithGlobals()` retrieves merged options from root + subcommand; cast to `{ node: string }` for type safety
- In beforeAll with xrpl.js WebSocket + CLI calls: use `execAsync` (promisify(exec)) NOT `spawnSync` to avoid blocking the event loop and breaking WebSocket connections
- After submitting a tx, `await new Promise(res => setTimeout(res, 8000))` is reliable for waiting for testnet validation
- DeletableAccounts amendment: new XRPL accounts start with sequence = current_ledger_index; AccountDelete requires sequence + 256 ≤ current_ledger → fresh testnet wallets always get tecTOO_SOON. Test actNotFound via Wallet.generate() (unfunded address) instead.

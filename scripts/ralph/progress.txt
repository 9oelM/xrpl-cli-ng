## Codebase Patterns
- `resolveWallet()` helper in escrow.ts is the canonical pattern for key material resolution — replicate it in new commands
- `--no-wait` option: NEVER pass `false` as default; use `.option("--no-wait", ...)` with no third arg; property is `options.wait` (bool); check `!options.wait`
- Credential ledger entry IDs: extract via `isCreatedNode(n) && n.CreatedNode.LedgerEntryType === "Credential"` then `n.CreatedNode.LedgerIndex`
- `convertStringToHex` from xrpl.js converts UTF-8 strings to hex for CredentialType / URI fields
- `isoTimeToRippleTime` from xrpl.js converts ISO8601 to XRPL epoch for Expiration fields
- CredentialCreate type is available in xrpl.js v4; Subject, CredentialType (hex), URI (hex), Expiration (XRPL epoch) are the fields

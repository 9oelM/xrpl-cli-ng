## Codebase Patterns
- MPTokenIssuanceID is 24 bytes (48 hex chars) = 4-byte big-endian sequence + 20-byte account ID (computed via decodeAccountID from xrpl)
- For IOU E2E tests, use direct issuance (issuer sends to holder) rather than rippling between two holders — avoids Default Ripple and lsfNoRipple trust line flag complexities
- MPTokenIssuance ledger entry type has NO MPTokenIssuanceID field; the ID must be computed from the creating tx's Sequence and Account
- For MPT payments, receiver must create MPTokenAuthorize before receiving MPT; the issuance must have tfMPTCanTransfer (flag 32) set
- Tests use vitest with `describe`/`it`/`expect` from "vitest"
- All imports use `.js` extension (ESM module project with `"type": "module"`)
- Test files live alongside source in `src/` for unit tests; E2E tests in `tests/e2e/`
- `npm run typecheck` for type checking, `npm test` for all tests
- Use `BigInt` for XRP drops arithmetic to avoid float precision issues

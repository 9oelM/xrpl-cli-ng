## Codebase Patterns
- MPToken tests use testnet (MPTokensV1 is live on testnet)
- MPTokenIssuanceID is available in `txResult.meta.mpt_issuance_id` after create (MPTokenIssuanceCreateMetadata type)
- Fallback: extract from AffectedNodes where LedgerEntryType === "MPTokenIssuance"
- `ledger_entry` request uses `mpt_issuance: issuanceId` (string) for MPT issuance lookup
- `account_objects` type for MPT issuances: `"mpt_issuance"`; for holder balances: `"mptoken"`
- MPTokenIssuanceSet uses `Holder` field (not `MPTokenHolder`); same for MPTokenAuthorize
- MPTokenIssuanceSetFlags: tfMPTLock=1, tfMPTUnlock=2
- MPTokenAuthorizeFlags: tfMPTUnauthorize=1
- Tests use `fundFromFaucet` from `tests/helpers/testnet.ts` (not a ticket-based fund.ts helper)
- Multiple issuance creates from same wallet are fine — each gets a unique issuanceId

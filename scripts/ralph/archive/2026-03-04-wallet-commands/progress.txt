## Codebase Patterns
- Node.js built-in `crypto` module used for AES-256-GCM encryption (createCipheriv/createDecipheriv) and PBKDF2 key derivation
- PATH must include `/home/vscode/.fnm/node-versions/v22.22.0/installation/bin` to use node/npm/npx/vitest
- Project uses ESM (`"type": "module"`) — imports must use `.js` extensions
- Test runner: `vitest run <file>` to run individual test files
- Typecheck: `npx tsc --noEmit`
- xrpl `ECDSA` enum cannot be imported as a named export in ESM context; use `import type { ECDSA }` for types only and cast string values: `"ecdsa-secp256k1"` (secp256k1) and `"ed25519"` (ed25519)
- E2E tests use `spawnSync` from Node.js `child_process` to run `tsx src/index.ts <args>`; PATH must be set in the spawn env
- `@scure/bip39` is a transitive dep of xrpl.js — use `generateMnemonic(wordlist, 128)` from `@scure/bip39` + wordlist from `@scure/bip39/wordlists/english` for 12-word BIP39 mnemonics; no extra install needed
- `Wallet.fromMnemonic(mnemonic, { mnemonicEncoding: 'bip39', derivationPath, algorithm })` is the xrpl.js API for BIP39 wallet derivation
- **CRITICAL**: `Wallet.fromSeed(seed)` defaults to `ed25519` algorithm regardless of seed type — use `deriveKeypair(seed)` from `ripple-keypairs` to correctly infer algorithm from seed encoding
- To derive address from seed without specifying algorithm: `const {publicKey} = deriveKeypair(seed); const address = deriveAddress(publicKey)` from `ripple-keypairs`
- Private key derivation from raw hex: ed25519 keys have "ED" prefix → use `@noble/curves/ed25519` `ed25519.getPublicKey(rawBytes)`; secp256k1 keys have "00" prefix → use `@noble/curves/secp256k1` `secp256k1.getPublicKey(rawBytes, true)`
- `@noble/curves` and `ripple-keypairs` are available as transitive deps of xrpl.js — no extra install needed
